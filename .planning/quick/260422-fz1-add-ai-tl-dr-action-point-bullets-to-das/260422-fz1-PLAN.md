# fz1 — AI TL;DR action-point bullets

Extend the existing nightly Haiku tool-use call in `supabase/functions/generate-insight/` with a third structured field `action_points: string[]` (2–3 bullets). Store in a new `action_points TEXT[]` column on `public.insights`, surface via `insights_v`, render as a `<ul>` inside `InsightCard.svelte` above the existing body. Reuse every existing safety mechanism (tool-use schema, digit-guard, deterministic fallback, SSR loader). Zero new dependencies, zero new Edge Function env vars.

## Must-haves

- `public.insights.action_points` column exists with `NOT NULL DEFAULT '{}'`
- `insights_v` view exposes the new column (same `security_invoker = true` pattern)
- Haiku tool schema requires `action_points` with `minItems: 2, maxItems: 3`
- Digit-guard rejects any bullet containing a number not in payload
- Fallback branch produces 3 deterministic bullets from existing 7-scalar input
- `InsightCard.svelte` renders bullets with dot glyph matching the existing `· auto-generated` style
- SSR loader at `src/routes/+page.server.ts` selects and passes through the new column

## Constraints honored

- Voice contract preserved: bullets are **observational focus areas**, not prescriptive advice. Existing FORBIDDEN PHRASINGS inherited; additional guard: "no You/We/Your/Let's/Should/Must in bullets".
- Digit-guard runs bullet-by-bullet against the same `allowed` set (reuses `flattenNumbers(payload)` — no new traversal).
- Fallback never fails open: if LLM produces bad shape or bad numbers, deterministic bullets from `deriveFallbackInput` scalars (guaranteed payload-valid).
- No schema churn: `TEXT[]` column on existing table; no new table, no new RPC.
- `CREATE OR REPLACE VIEW` — zero downtime.
- Mobile-first: bullets cap at 60 chars × 3, tested at 375 px in QA.

### 1. Migration — `supabase/migrations/0035_insights_action_points.sql`

```sql
-- 0035_insights_action_points.sql
-- Adds action_points TEXT[] column to public.insights and refreshes insights_v.
-- Bullets are written by the nightly generate-insight Edge Function (Haiku tool-use)
-- and rendered by the dashboard InsightCard. Default '{}' backfills existing rows.

ALTER TABLE public.insights
  ADD COLUMN IF NOT EXISTS action_points TEXT[] NOT NULL DEFAULT '{}';

CREATE OR REPLACE VIEW public.insights_v
  WITH (security_invoker = true) AS
SELECT
  id,
  restaurant_id,
  business_date,
  generated_at,
  headline,
  body,
  action_points,
  model,
  fallback_used
FROM public.insights
WHERE restaurant_id::text = (auth.jwt()->>'restaurant_id');
```
