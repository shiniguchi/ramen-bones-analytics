<objective>
Fix the silent dashboard crash where Cohort Revenue + Cohort Avg LTV cards throw `RangeError: Invalid time value` under month grain, causing reactive `$derived` to crash and all KPI tiles to fall back to `0 €`.

Purpose: Unblock the dashboard in production. 3 identical stack traces are live on https://ramen-bones-analytics.pages.dev. Root cause: DB `customer_ltv_mv.cohort_month` is Postgres DATE → returns `'YYYY-MM-DD'`; `pickCohortKey` returns it raw; `formatBucketLabel` appends `'-01'` expecting `'YYYY-MM'` input; `parseISO('2025-06-01-01')` → Invalid Date → format throws.

Output: 1-line fix in `cohortAgg.ts` + regression tests that pin the contract. Bug cannot silently return.
</objective>

export function formatBucketLabel(bucket: string, grain: 'day' | 'week' | 'month'): string {
  return grain === 'month'
    ? format(parseISO(bucket + '-01'), 'MMM')   // EXPECTS 'YYYY-MM' — adds '-01' → must be length 7
    : format(parseISO(bucket), 'MMM d');
}
