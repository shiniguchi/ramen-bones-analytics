# Quick Task 260428-c87 — Summary

## What changed

- **`scripts/ingest/refresh.ts`** — added a top-of-function guard in `refreshAndMaybeTriggerInsight` that returns a no-op `RefreshResult` when `process.env.VITEST === 'true'`. Vitest sets this env var automatically; production CLI runs do not.

## Why

Quick task `260428-wmd`'s post-ingest hook was triggering both:
- An RPC call to `refresh_analytics_mvs` against TEST Supabase (side effect on a non-DEV project), and
- A POST to `${TEST_SUPABASE_URL}/functions/v1/generate-insight` — the Edge Function only deploys to DEV, so this returned `404 NOT_FOUND`. Suppressed by the existing `try/catch` but logged as `post_ingest_error` to stderr, polluting test output.

## Verification

- `npx vitest run tests/ingest/loader.test.ts tests/ingest/idempotency.test.ts` — **5/5 pass**, 0 hits for `post_ingest_error`, 0 hits for `generate-insight call failed` (was ≥3 hits across the two files pre-fix).
- `npm run ingest -- --dry-run` from CLI — produces the existing report line, no environment-detection regressions. (Real-run path was already verified earlier in session — VITEST env is unset under tsx, so the guard is a no-op there.)

- Other CI flakes unrelated to the post-ingest hook (Issue B / TEST_SUPABASE_URL secret remains user-action).
