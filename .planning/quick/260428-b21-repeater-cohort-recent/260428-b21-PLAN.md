# Quick Task 260428-b21 — Show recent cohorts on the repeater chart

## Why

The repeater cohort chart's rightmost bar is the cohort starting **2026-04-06** even though data exists through 2026-04-27. The cause is `cohortRepeaterCountByVisitBucket` (`src/lib/cohortAgg.ts:128`) filtering cohorts with `total_repeaters < SPARSE_MIN_COHORT_SIZE` (= 5). The Apr 13 / 20 / 27 cohorts haven't had time to accumulate 5+ repeat customers, so they get dropped.

Per user direction (Q1 → "yes for sake of simplicity"): lower this filter so recent cohorts show up. They'll naturally render as short bars — exactly the visual signal the user wants ("the latest cohorts are still maturing"), without adding a separate badge / opacity marker / tooltip. Bars with zero repeaters just won't render (stack height = 0), which is fine.

## Tasks

### 1. Lower the threshold in `src/lib/cohortAgg.ts:128`

Change:
```ts
.filter(([, v]) => v.total_repeaters >= SPARSE_MIN_COHORT_SIZE)
```
to:
```ts
.filter(([, v]) => v.total_repeaters >= 1)
```

Replace the comment immediately above (line 47-48 in the JSDoc) so future readers know why this chart deviates from the shared `SPARSE_MIN_COHORT_SIZE` convention.

**Verify:** `grep "SPARSE_MIN_COHORT_SIZE" src/lib/cohortAgg.ts` returns no matches in the function body (the import may stay if used elsewhere — actually it's not used elsewhere in this file, so the import line can also be dropped if it becomes unused).

### 2. Verify on localhost

Reload `localhost:5173/?range=all&grain=week`, scroll to "初回来店グループ別のリピーター", scroll the chart horizontally to the right edge, confirm cohorts past **2026-04-06** are now visible (likely 2026-04-13 with a few repeaters; 2026-04-20 maybe; 2026-04-27 will still be absent because zero repeaters).

**Verify:** rightmost bar's tooltip date is later than 2026-04-06.

## must_haves

- The repeater cohort chart shows cohorts past 2026-04-06 (specifically Apr 13 if it has ≥1 repeater).
- Other charts that use `SPARSE_MIN_COHORT_SIZE` (e.g., CohortRetentionCard) are unaffected.

## Out of scope

- Re-tuning the threshold for other charts.
- Changing the constant `SPARSE_MIN_COHORT_SIZE` itself (kept at 5 for the retention chart's sparse-filter, which has different semantics).
