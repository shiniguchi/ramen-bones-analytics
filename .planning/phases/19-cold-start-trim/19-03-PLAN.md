# Plan 19-03: i18n per-locale dynamic imports

**Goal:** Cut the 76 KB `messages.ts` monolith from the cold bundle. Split into 5 per-locale dict files (~15 KB each); keep `en` eager (fallback + `MessageKey` inference); lazy-import `de/ja/es/fr` via `loadDict(locale)`. Zero changes to the 19 component call sites (`t(locale, key)`).

**Phase:** 19 (Cold-Start Trim)
**Status:** Pending
**Effort:** ~2 hours
**Depends on:** Nothing (independent of 19-01 / 19-02)
**Blocks:** 19-04 (final QA)

## Pillar audit

- **Minimal** — `loadDict()` is a single function with a module-level `Map`. No new class, no new context primitive. `en` stays eager (it's already a dependency of `MessageKey`). Adding locale `fr` later = one new file.
- **Scalable** — new locale = 1 dict file, 0 KB cold-bundle cost. Module-level `Map` cache means each locale downloads at most once per session.
- **Dynamic** — locale is runtime-resolved via `event.cookies.get(LOCALE_COOKIE)` (already wired in `hooks.server.ts:14-15`). No hardcoding. Vite auto-splits `import('./dict/${locale}.ts')` into one chunk per locale.
- **Universal** — `seedDict(locale, dict)` in `+layout.svelte` mirrors `clientFetch`'s Map cache pattern (`src/lib/clientFetch.ts:13`). `t()` signature unchanged; 19 call sites untouched.

## Implementation steps

### Step 1 — Create the 5 dict files

For each locale `{en,de,ja,es,fr}`, create `src/lib/i18n/dict/<locale>.ts`:

```ts
// src/lib/i18n/dict/en.ts
const en = {
  // … paste the contents of the current `const en = { … }` block verbatim …
} as const;

export default en;
```

The `de`, `ja`, `es`, `fr` files follow the same pattern: `export default { … } as const;` (drop the `Record<MessageKey, string>` annotation — TypeScript will infer it from the object literal, which is enough for the cache).

### Step 2 — Rewrite `messages.ts`

Replace the entire current 1358-line file with:

```ts
// src/lib/i18n/messages.ts
// Phase 19-03: en imported eagerly (always needed for MessageKey + fallback).
// Other locales lazy-loaded via loadDict() — one chunk per locale emitted by Vite.
import type { Locale } from './locales';
import en from './dict/en';

export type MessageKey = keyof typeof en;

type Dict = Record<MessageKey, string>;

const dictCache = new Map<Locale, Dict>();
dictCache.set('en', en as Dict);

export async function loadDict(locale: Locale): Promise<void> {
  if (dictCache.has(locale)) return;
  // Dynamic import — Vite emits one async chunk per locale.
  const m = await import(`./dict/${locale}.ts`);
  dictCache.set(locale, m.default as Dict);
}

// Synchronous cache seed — called by +layout.svelte to hydrate the client
// from the SSR payload without a second round-trip.
export function seedDict(locale: Locale, dict: Dict): void {
  dictCache.set(locale, dict);
}

// Exported for +layout.server.ts to include the dict in the SSR payload.
export function getDict(locale: Locale): Dict {
  return dictCache.get(locale) ?? (en as Dict);
}

function interpolate(template: string, vars?: Record<string, string | number>): string {
  if (!vars) return template;
  return template.replace(/\{(\w+)\}/g, (_, k) => k in vars ? String(vars[k]) : `{${k}}`);
}

/**
 * Look up a UI string. Falls back to en if locale dict not loaded or key missing.
 * API unchanged — 19 call sites unmodified.
 */
export function t(
  locale: Locale,
  key: MessageKey,
  vars?: Record<string, string | number>
): string {
  const raw = dictCache.get(locale)?.[key] ?? en[key as keyof typeof en] ?? key;
  return interpolate(raw, vars);
}
```

Remove the `messages` export (`export const messages = { en, de, ja, es, fr }`) — it was only consumed by `t()` internally.

### Step 3 — Seed in `hooks.server.ts`

After `event.locals.locale` is resolved (line 15), add one line:

```ts
import { loadDict } from '$lib/i18n/messages';
// …
event.locals.locale = isLocale(cookieLocale) ? cookieLocale : DEFAULT_LOCALE;
await loadDict(event.locals.locale); // populates dictCache for this locale before SSR
```

### Step 4 — Return `dict` from `+layout.server.ts`

Read the current `+layout.server.ts`. On every return path (PUBLIC_PATHS, E2E, normal), add `dict: getDict(locals.locale ?? DEFAULT_LOCALE)`:

```ts
import { getDict } from '$lib/i18n/messages';
// …
return {
  restaurantId,
  locale,
  dict: getDict(locale)
};
```

This serialises the ~15 KB dict object into the SSR payload once, so client hydration is synchronous and there is no flash.

### Step 5 — Seed dict in `+layout.svelte`

In the `<script>` block (not inside `$effect` — must run synchronously before children render):

```ts
import { seedDict } from '$lib/i18n/messages';
let { data, children } = $props();
seedDict(data.locale, data.dict);
```

`seedDict` on the server is a no-op (the cache is already populated by `await loadDict()` in hooks). On the client it runs during component initialisation, before any child `t()` call, so no flash-of-english on locale-switch.
