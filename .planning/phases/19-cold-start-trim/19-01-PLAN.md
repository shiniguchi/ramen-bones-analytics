# Plan 19-01: LazyMount loader prop + 5-card defer

**Goal:** Cut the eager-mount surface in `+page.svelte` from 5 chart cards to 0. Extend `LazyMount` with a single optional `loader` prop so chart-card modules (and their LayerChart + d3 + sibling-component transitive imports) only download when the card scrolls into view.

**Phase:** 19 (Cold-Start Trim)
**Status:** Pending
**Effort:** ~3 hours
**Depends on:** Nothing (first plan)
**Blocks:** 19-02 (consumer cards must be lazy before SSR queries can be deferred)

---

## Pillar audit

- **Minimal** — extend existing `LazyMount` with one prop; do not introduce a second lazy primitive. No `manualChunks`. No new abstractions. Loader is a thunk, not a config object.
- **Scalable** — every future chart card uses the same `loader={() => import('…')}` form. Adding card #11 = zero cold-bundle cost.
- **Dynamic** — `loader` is a function, not a static string. Vite's import-graph analysis emits one async chunk per dynamic import automatically.
- **Universal** — `LazyMount.svelte:8-12` "single mandated lazy idiom" rule preserved. Existing children-snippet API for non-deferred uses still works.

## Implementation steps

### Step 1 — Extend `LazyMount.svelte`

**File:** `src/lib/components/LazyMount.svelte`

Add to `$props` block (currently lines 25-35):

```svelte
let {
  minHeight = '240px',
  rootMargin = '200px',
  onvisible,
  loader,
  props = {},
  children
}: {
  minHeight?: string;
  rootMargin?: string;
  onvisible?: () => void;
  loader?: () => Promise<{ default: import('svelte').Component }>;
  props?: Record<string, unknown>;
  children?: import('svelte').Snippet;
} = $props();
```

Add reactive state:

```ts
let Loaded = $state<import('svelte').Component | null>(null);
```

Modify the IntersectionObserver effect (currently lines 40-60) so that when `loader` is set, the visible callback awaits the import and binds the default export:

```ts
$effect(() => {
  if (!sentinel || mounted) return;
  if (typeof IntersectionObserver === 'undefined') {
    mounted = true;
    if (loader) loader().then((m) => (Loaded = m.default));
    onvisible?.();
    return;
  }
  const io = new IntersectionObserver(
    (entries) => {
      if (entries.some((e) => e.isIntersecting)) {
        mounted = true;
        if (loader) loader().then((m) => (Loaded = m.default));
        onvisible?.();
        io.disconnect();
      }
    },
    { rootMargin }
  );
  io.observe(sentinel);
  return () => io.disconnect();
});
```

Modify the render block (currently lines 63-70). Three branches:

```svelte
<div bind:this={sentinel} style:min-height={minHeight} class="w-full">
  {#if !mounted}
    <div class="animate-pulse bg-neutral-100 rounded-lg" style:min-height={minHeight}></div>
  {:else if loader}
    {#if Loaded}
      <svelte:component this={Loaded} {...props} />
    {:else}
      <div class="animate-pulse bg-neutral-100 rounded-lg" style:min-height={minHeight}></div>
    {/if}
  {:else if children}
    {@render children()}
  {/if}
</div>
```

The "single idiom" header comment (lines 8-12) gets one extra line:

```
//   Two ways to render:
//     1. Pass `children` snippet (existing behaviour).
//     2. Pass `loader: () => import('./Card.svelte')` for dynamic-import.
//   Pick exactly ONE; mutually exclusive.
```

Their existing `onvisible` callbacks (`loadDailyKpi`, `loadRetention`, etc.) stay — `loader` and `onvisible` are independent props.

Example for the existing DailyHeatmap site:

```svelte
<!-- Before (snippet form) -->
<LazyMount minHeight="280px" onvisible={loadDailyKpi}>
  {#snippet children()}
    <DailyHeatmapCard data={dailyKpi} />
  {/snippet}
</LazyMount>

<!-- After (loader form) -->
<LazyMount
  minHeight="280px"
  onvisible={loadDailyKpi}
  loader={() => import('$lib/components/DailyHeatmapCard.svelte')}
  props={{ data: dailyKpi }}
/>
```

**Reactive `props`:** because `dailyKpi`, `retention`, `customerLtv`, `repeaterTx`, `retentionMonthly`, `monthsOfHistory` are `$state` rune values, passing them through `props={{ … }}` re-renders the loaded component on every change. No special wiring needed.
