# Phase 4 вҖ” UI Design Contract

> Visual and interaction contract for the mobile reader dashboard. Generated by gsd-ui-researcher from CONTEXT.md (D-01..D-22) + CLAUDE.md stack lock. Verified by gsd-ui-checker.

**Target viewport:** 375px baseline (iPhone SE / mini). Single-column card stream. Desktop is "single column works wider"; not a separate design.

**Requirements covered:** UI-01..UI-11.

## Spacing Scale

8-point scale, multiples of 4. Tailwind v4 default spacing scale is used verbatim; no custom spacing tokens.

| Token | Value | Tailwind | Usage |
|-------|-------|----------|-------|
| xs | 4px | `p-1` / `gap-1` | Icon-to-label gaps inside buttons/chips |
| sm | 8px | `p-2` / `gap-2` | Inside-card element spacing (KPI label вҶ’ number) |
| md | 16px | `p-4` / `gap-4` | Default card interior padding; inter-element within a card |
| lg | 24px | `p-6` / `gap-6` | Between-card vertical spacing in the scroll stream |
| xl | 32px | `p-8` / `gap-8` | Header-to-first-card, footer padding |
| 2xl | 48px | `p-12` | Not used in v1 (reserved) |

**Touch-target exception:** All tappable elements (chips, grain toggle, logout glyph, chart points) MUST be вүҘ44x44px even if visual size is smaller. Hit area via `min-h-11 min-w-11` or invisible padding. Mandatory for mobile.

**Page-level horizontal padding:** `px-4` (16px) on the scroll container at 375px. No horizontal scroll ever. Content max-width `max-w-screen-sm` (640px) centered on wider viewports.

**Card corner radius:** `rounded-xl` (12px) on all cards вҖ” distinguishes cards from the background without feeling playful.

## Typography

Exactly 4 sizes, exactly 2 weights. Line heights specified.

| Role | Size | Weight | Line Height | Tailwind | Usage |
|------|------|--------|-------------|----------|-------|
| Caption | 12px | 400 (regular) | 1.4 | `text-xs leading-[1.4] font-normal` | Freshness label, "vs prior 7d" delta caption, LTV data-depth footer (italic), sparse-cohort hint, chart axis labels |
| Body | 14px | 400 (regular) | 1.5 | `text-sm leading-normal font-normal` | Card titles, chip labels, tooltip content, empty-state copy, legend |
| Heading | 20px | 600 (semibold) | 1.2 | `text-xl leading-tight font-semibold` | Card section headers ("Cohort retention", "LTV-to-date"), header brand name |
| Display | 32px | 600 (semibold) | 1.1 | `text-[32px] leading-[1.1] font-semibold tabular-nums` | KPI tile big numbers (`вӮ¬ 4,280`), LTV bar value labels |

**Rules:**
- **Only 400 and 600 ship.** No 500, no 700. Enforced in review.
- **`tabular-nums` mandatory** on all numeric displays (KPI tiles, bar values, deltas, tooltips). Prevents digit-width jitter when numbers update.
- **Delta caption** (`в–І +12% vs prior 7d`) uses the Caption role with a color (see Color В§). The arrow glyph is part of the caption string.
- **Italic** reserved exclusively for the LTV data-depth footer (D-17). No other italic in v1.
- **Font size floor 12px** вҖ” nothing smaller, ever. Legibility on phone + 44-year-old friend's eyes.

**Accent NOT used for:**

**LTV bar color:** single `#2563eb` (blue-600) at 85% opacity вҖ” same accent, single-hue because bars don't need differentiation.

## Interaction Contracts

### Date-range chips (D-04, D-05, D-19a)

- 5 chips: `Today` / `7d` / `30d` / `90d` / `All`. Default `7d`.
- State lives in URL `?range=7d`. SSR-readable. Shareable.
- Active chip: `bg-blue-600 text-white`. Inactive: `bg-white border border-zinc-200 text-zinc-900`.
- Sticky to top of viewport on scroll (`sticky top-0 z-10`) with `bg-zinc-50/95 backdrop-blur`.
- Affects: `Transactions` card, `Avg ticket` card, `New vs returning` card (the D-19a exception).
- Does NOT affect: 3 fixed revenue tiles, cohort retention card, LTV card, frequency card.
- Tap target: 44px min height each.

### Cohort grain toggle (D-12)

- Segmented 3-way: `Day | Week | Month`. Default `Week`.
- Lives inside `CohortRetentionCard.svelte` header row, right-aligned.
- State via `?grain=week` in URL (second URL param). Shared with LTV card per D-16.
- Active segment uses accent color (`text-blue-600 bg-blue-50`).
- Tap target: 44px min height per segment.

### LayerChart tap tooltip (D-15)

- Mobile: tap any point вҶ’ tooltip pins until next tap elsewhere.
- No hover state. Pointer-events: none on decorative elements.
- Tooltip content: `cohort_start` (formatted `MMM d`), `period` (`Week N`), `retention_rate` (`NN%`), `cohort_size` (`N customers`).
- Tooltip uses shadcn `tooltip` primitive, positioned above the tap point, 8px offset.
- Tooltip background: `bg-zinc-900 text-white text-sm p-2 rounded-md`.

### Logout (D-03)

- Icon button, top-right header. `lucide-svelte` `LogOut` icon at 20px, inside a 44px hit target.
- Form POST to `+page.server.ts` logout action. No client-side confirmation.
- Redirects to `/login` on success.

### Focus management

- All interactive elements (chips, grain toggle, buttons, chart points) get `focus-visible:ring-2 focus-visible:ring-blue-600 focus-visible:ring-offset-2 focus-visible:ring-offset-zinc-50`.
- Tab order follows visual order top-to-bottom.
- `/login` focuses email input on mount.

## Accessibility Contract

- **Contrast:** Every text/background pair MUST meet WCAG AA (4.5:1 for body, 3:1 for large 20px+). Verified: `zinc-900` on `white` = 16.1:1, `zinc-500` on `zinc-50` = 4.6:1, `white` on `blue-600` = 8.6:1, `green-600` on `white` = 3.9:1 вҖ” FAILS AA for small text; **delta captions using green/red MUST be at вүҘ14px not 12px, or darken to `green-700` (4.5:1) / `red-700` (4.8:1).** Use `green-700` and `red-700` for delta captions.

**Correction applied to Color table:** delta captions use `text-green-700` (`#15803d`) and `text-red-700` (`#b91c1c`), not `-600`. Replace above.

- **Touch targets:** 44x44px minimum on every interactive element.
- **Aria:** `aria-label` on icon-only buttons (logout, any icon in chart tooltips). `aria-current="true"` on active chip. `role="group"` + `aria-label="Date range"` on chip bar. `role="radiogroup"` on grain toggle.
- **Semantic HTML:** `<header>`, `<main>`, `<section>` per card, `<h2>` for each card title. Chart has an `<figcaption>` fallback describing the data textually for screen readers.
- **Reduced motion:** `@media (prefers-reduced-motion)` disables LayerChart entrance animations.
- **Keyboard:** Full keyboard navigation. Chips = arrow keys between, Enter/Space to activate. Grain toggle = arrow keys between segments.

## Viewport Contract (UI-02, UI-11)

- **Baseline:** 375px width (iPhone SE / mini / 12 mini). Every PR verified at this width before merge.
- **No horizontal scroll ever.** Enforced by `overflow-x-hidden` on `<body>` and per-card content boundaries.
- **Card stack:** single column, `gap-6` between cards, `px-4` page padding. Max content width `max-w-screen-sm` centered on wider viewports.
- **Sticky elements:** only the chip bar. Header scrolls away.
- **Chart width:** charts use `width: 100%` of card interior (`card - md md` = `375 - 16 - 16 - 16 - 16 = 311px` usable). LayerChart responsive.
- **Verification:** PR reviewer opens the CF Pages preview URL in Chrome DevTools device mode at iPhone SE (375Г—667) and captures one screenshot. Optional stretch: `scripts/verify-viewport.mjs` Playwright screenshot (gsd-planner decides).
