Create `scripts/seed-recent-transactions.sql`. Requirements:

1. **Idempotent**: Use `source_tx_id` prefix `demo-recent-` and `ON CONFLICT (restaurant_id, source_tx_id) DO NOTHING`. Running the script twice must produce the same DB state.

2. **Date range**: last 14 business_days relative to `CURRENT_DATE AT TIME ZONE 'Europe/Berlin'`. Use `generate_series(CURRENT_DATE - INTERVAL '14 days', CURRENT_DATE, '1 day'::interval)`.

3. **Row count**: ≥50 transactions total. Distribute roughly 3-5 per day across the 14 days (so kpi_daily_v has data for today / 7d / 30d windows).

4. **Cohort signal**: Use ~8 distinct `card_hash` values so `cohort_mv` gets multiple customers. Stagger first-visits across 2+ weekly cohorts: ~4 card_hashes first appear in week -2, ~4 in week -1. Several card_hashes should make repeat visits across days so retention + new-vs-returning + frequency have real numbers.

5. **Non-zero numerics**:
   - `gross_cents` between 1500 and 4500 (€15–45 per transaction, realistic ramen ticket)
   - `net_cents` = round(gross_cents / 1.07) (7% food tax — matches Phase 02-04 semantics)
   - `tip_cents` between 0 and 500 (mix)
   - `payment_method` mix: ~70% 'card', ~30% 'cash'
   - Cash rows get `card_hash = NULL` (excluded from cohort — matches ING-04)
   - Card rows get a deterministic synthetic `card_hash` (e.g. `encode(sha256(('demo-card-' || card_num)::bytea), 'hex')`)

6. **restaurant_id**: hardcode `'ba1bf707-aae9-46a9-8166-4b6459e6c2fd'::uuid`.

```sql
-- scripts/seed-recent-transactions.sql
-- Idempotent seed: ≥50 synthetic transactions across the last 14 business_days
-- for restaurant ba1bf707-aae9-46a9-8166-4b6459e6c2fd. Closes Gap 3 from
-- 05-HUMAN-UAT.md (no real data due to April 2026 Worldline blackout).

INSERT INTO public.transactions (
  restaurant_id, source_tx_id, occurred_at, business_date,
  gross_cents, net_cents, tip_cents, payment_method, card_hash
)
SELECT
  'ba1bf707-aae9-46a9-8166-4b6459e6c2fd'::uuid,
  'demo-recent-' || d::date || '-' || i,
  (d + (8 + i) * INTERVAL '1 hour')::timestamptz,  -- occurred_at spread across lunch/dinner
  d::date,
  (1500 + (random() * 3000)::int),                 -- gross_cents 1500-4500
  round((1500 + (random() * 3000)::int) / 1.07)::int,  -- net_cents
  (random() * 500)::int,                           -- tip_cents 0-500
  CASE WHEN random() < 0.7 THEN 'card' ELSE 'cash' END,
  CASE
    WHEN random() < 0.7 THEN encode(sha256(('demo-card-' || ((i + extract(day from d)::int) % 8))::bytea), 'hex')
    ELSE NULL
  END
FROM generate_series(CURRENT_DATE - INTERVAL '14 days', CURRENT_DATE, '1 day'::interval) d,
     generate_series(1, 5) i  -- 5 rows per day × 15 days = 75 rows
ON CONFLICT (restaurant_id, source_tx_id) DO NOTHING;
```
