Create `.cloudflare/pages-project.md` recording both URLs + the deploy timestamp:

```markdown
# Cloudflare Pages deployment

**Project:** ramen-bones-analytics
**Production URL:** https://ramen-bones-analytics.pages.dev
**First deploy:** <ISO timestamp>
**Deploy command:** `npx wrangler pages deploy .svelte-kit/cloudflare --project-name=ramen-bones-analytics --branch=main --commit-dirty=true`

## Env vars (bound in CF Pages dashboard → Settings → Environment variables → Production)
- PUBLIC_SUPABASE_URL
- PUBLIC_SUPABASE_PUBLISHABLE_KEY

## Redeploy
Run `npm run build && npx wrangler pages deploy .svelte-kit/cloudflare --project-name=ramen-bones-analytics --branch=main --commit-dirty=true`
```

Then smoke-test the URL:

```bash
curl -sI https://ramen-bones-analytics.pages.dev/ | head -1
curl -sL https://ramen-bones-analytics.pages.dev/login | grep -qi "sign in\|login\|email"
```

First curl must print `HTTP/2 200` (or HTTP/1.1 200). Second must exit 0 (login page renders).

If deploy fails with "env var not set" errors for PUBLIC_SUPABASE_URL at runtime (visible in `wrangler pages deployment tail`), return to Task 2 — the env vars were not bound correctly.
