## Risk 2 disposition — Prophet path chosen

**Path A taken** — window_start_for_grain + train_end wired into prophet_fit.py at both pred_dates_for_grain call sites.

Evidence: Reading prophet_fit.py end-to-end showed the cf-track (line 318) and bau-track (line 357) `pred_dates_for_grain` calls feed their returned date list into a `_build_future_df(pred_dates, …)` DataFrame which is then passed to `m.predictive_samples(future_df)`. Prophet's `predictive_samples()` accepts arbitrary `ds` columns including past dates and emits `yhat` values that follow the trained seasonality+trend curve projected onto those calendar dates — semantically equivalent to backtest values for the past-window portion. No need for a custom cross-validation harness; the existing predict path is sufficient. The single `samples.shape == (horizon, N_PATHS)` assertion was the only invariant that had to flex (now `(len(pred_dates), N_PATHS)`).

Acceptance:
- `grep -c "window_start_for_grain" scripts/forecast/prophet_fit.py` = **3** (1 import + 2 call sites) — Path A satisfied.
