## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 3 - Blocking] TypeScript narrowing failure on dynamic i18n key**
- **Found during:** Task 4 (CalendarRevenueCard D-16 tooltip extension)
- **Issue:** `t(page.data.locale, \`forecast_model_${name}\`)` produces a template literal type `\`forecast_model_${string}\`` which does not narrow to the MessageKey union → svelte-check error: `Argument of type '\`forecast_model_${string}\`' is not assignable to parameter of type '"brand_name" | ... | "forecast_model_naive_dow"'`.
- **Fix:** Imported `MessageKey` type from `$lib/i18n/messages` and added `as MessageKey` cast at the call site in both CalendarRevenueCard.svelte (Task 4) and preemptively in CalendarCountsCard.svelte (Task 5).
- **Files modified:** `src/lib/components/CalendarRevenueCard.svelte`, `src/lib/components/CalendarCountsCard.svelte`
- **Verification:** `npm run check` returns to baseline 7 errors (was 8 with the new error) → no new errors introduced; `npm run build` green.
- **Committed in:** `0695056` (Task 4) and `151b5c6` (Task 5).

**2. [Rule 2 - Missing Critical] pastForecastBuckets added to totalSlots on BOTH cards**
- **Found during:** Task 2 (CalendarRevenueCard chartXDomain widening planning)
- **Issue:** Plan instructs widening chartXDomain (LEFT edge) but does not call out feeding `pastForecastBuckets` into `totalSlots`. Without that, `chartW = computeChartWidth(totalSlots, cardW)` would NOT account for the LEFT extension and the chart canvas would compress bars proportionally to fit the wider domain in the same width → bars become unreadable on mobile when window_start is far back.
- **Fix:** `totalSlots = chartData.length + fcDates.size + pastForecastBuckets` on both cards. Pre-D-15: `pastForecastBuckets=0` → identical math.
- **Files modified:** `src/lib/components/CalendarRevenueCard.svelte`, `src/lib/components/CalendarCountsCard.svelte`
- **Verification:** Pre-D-15 path verified by code reading (forecastWindowStart=null → pastForecastBuckets=0 → totalSlots unchanged); post-D-15 path covered by Task 6 visual verification when re-driven.
- **Committed in:** `0918ecc` (Task 2) and `644e8a5` (Task 3).
