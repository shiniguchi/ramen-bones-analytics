## Decisions Made

- **`{@const}` hoisted to `$derived` in script block.** Svelte 5 forbids `{@const}` as a non-block-immediate child of `<div>`. Auto-fixed [Rule 1 - Bug] by computing `isCIOverlap = $derived(ciOverlapsZero || (headline?.row.cumulative_uplift_eur ?? 0) === 0)` in the script. Cleaner than restructuring the template into nested `{#if}` blocks.
- **`MessageKey` type cast on `heroKey`.** TypeScript template literal types do not narrow `\`uplift_hero_${tier}_${sign}\`` to the `MessageKey` union. Auto-fixed [Rule 3 - Blocking] via `as MessageKey` cast on the dynamic legs of the matrix and `$derived.by<MessageKey>(...)` on the wrapper. Established pattern from 16.1-01 (forecast_model_${name}). Runtime safety: if the resolved key is not in the dictionary, `t()` falls back to EN (per messages.ts:1041 area).
- **W4 Y-label above-Chart placement (NOT in-Svg Axis).** RESEARCH.md documented 375px clipping risk for axis-rotated labels at the mobile breakpoint; planner pre-locked the above-Chart `<p>` choice. Single-line render of the `<p>` (text + close tag inline) so the W4 grep gate (`uplift_sparkline_y_label` within 2 lines BEFORE `<Chart>`) passes.
- **Hero subtitle rendered in BOTH the empty-state branch AND the known-state branch.** Per RESEARCH.md edge case: subtitle gives context even when the counterfactual is still computing (otherwise the empty-state shell reads as orphaned).
- **Locale-aware date via `Intl.DateTimeFormat`.** Zero bundle cost; works in CF Workers runtime; no date-fns/locale loaders needed. Coexists with the existing date-fns `format()` call inside the sparkline tooltip header (`format(pt.date as Date, 'MMM d')` is locale-agnostic month abbreviation — acceptable per planner).
- **`format` and `parseISO` imports retained.** `parseISO` is reused by `formatHeadlineDate` AND the sparkline tooltip; `format` is reused by the sparkline tooltip header. Removing them would have broken the sparkline tooltip.
- **File header comment updated to reflect Phase 16.1-03 reality.** The pre-edit comment described the "honest CI overlaps zero label rule" — now obsolete because the plain-language hero replaces that mechanism. Updated comment notes the D-05..D-11 + D-18 redesign and that statistical detail lives INSIDE the disclosure panel. Side benefit: hardcoded-English regression guard returns ZERO (the comment was the only remaining hit).
