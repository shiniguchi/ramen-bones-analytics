# Phase 16.3 Plan 07: Wire-Charts Summary

EventBadgeStrip wired into the 5 date-axis charts identified in CONTEXT.md D-09 inventory (`CalendarRevenueCard`, `CalendarCountsCard`, `CalendarItemsCard`, `CalendarItemRevenueCard`, `RepeaterCohortCountCard`); `EventMarker.svelte` deleted (D-04 step 3) along with its sibling unit test.

## What Was Done

### Cards 1+2 — `CalendarRevenueCard` + `CalendarCountsCard` (forecast consumers)
- Added `rangeStartIso()` deriving `min(historical[0].bucket_d, overlay.forecastWindowStart)` formatted yyyy-MM-dd, threaded into `createForecastOverlay({ rangeStart: rangeStartIso })` (D-05).
- Read clamped events from `overlay.events` (Plan 16.3-06 getter).
- Computed `eventBuckets` (band-position math: `chartData.length × stripWidth`) — caller-owned scale per D-02.
- `stripWidth = chartW ?? cardW` falls back to clientWidth when `computeChartWidth` returns `undefined` (chart fits without scroll).
- Mounted `<EventBadgeStrip>` inside the existing `overflow-x-auto` scroll wrapper, immediately after `</Chart>` (scrolls in sync with bars).

### Cards 3+4 — `CalendarItemsCard` + `CalendarItemRevenueCard` (actuals-only)
- Standalone `clientFetch<{ events }>('/api/forecast?...&range_start=...')` reading **only** the events field (option a per CONTEXT.md). No `useEvents()` factory extracted — single-purpose call.
- Card's user-selected grain (`getFilters().grain`) propagated to `/api/forecast?granularity=...` so events match bucket boundaries across day/week/month.
- `bucket_iso` companion key added to `chartData` rows BEFORE `formatBucketLabel` replaces `bucket` with the display string — strip aligns events to raw ISO bucket dates.
- `lastFetchedKey = ${grain}|${rs}` guard prevents reactive loops on re-renders.
- Mounted `<EventBadgeStrip>` inside the chart's scroll wrapper.

### Card 5 — `RepeaterCohortCountCard` (cohort-week/month axis)
- Standalone `/api/forecast?...` fetch (option a — CONTEXT.md "Integration Points" §"RepeaterCohortCountCard events feed").
- `granularity={cohortGrain}` (week | month — D-17 day clamps to week).
- `cohort_iso` preserved in `chartData` (raw `yyyy-MM-dd` from `cohortAgg`) for strip alignment.
- Strip mounted with `grain={cohortGrain}`.

### Deletion — `EventMarker.svelte` (D-04 step 3)
- Component deleted via `git rm`.
- `tests/unit/EventMarker.test.ts` deleted (would break svelte-check — Rule 3 blocking-issue auto-fix).
- Three dangling comment-only references in `eventTypeColors.ts`, `forecastEventClamp.ts`, and `api/forecast/+server.ts` updated to remove `EventMarker` mentions (preserves historical context without dangling identifier names).

## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 3 — Blocking issue] `tests/unit/EventMarker.test.ts` deleted**
- **Found during:** Task 4 (`npm run check` after `git rm` of EventMarker.svelte)
- **Issue:** Test imported `../../src/lib/components/EventMarker.svelte`; svelte-check failed with "Cannot find module".
- **Fix:** Deleted the test file (component no longer exists, so unit-testing it is meaningless).
- **Files modified:** `tests/unit/EventMarker.test.ts` (deleted)
- **Commit:** `0a64603`

**2. [Rule 2 — Missing functionality] `bucket_iso` companion key**
- **Found during:** Task 2 + Task 3 (CalendarItems*, RepeaterCohortCount)
- **Issue:** Cards' `chartData` derivations replace the raw bucket key with a display label via `formatBucketLabel()` BEFORE the chart sees it. EventBadgeStrip needs the raw ISO to match `event.date`.
- **Fix:** Added `bucket_iso` (or `cohort_iso` for RepeaterCohortCountCard) companion key holding the raw ISO, derived BEFORE `formatBucketLabel` runs.
- **Files modified:** `CalendarItemsCard.svelte`, `CalendarItemRevenueCard.svelte`, `RepeaterCohortCountCard.svelte`
- **Commits:** `28006d9`, `cce88d5`

**3. [Rule 2 — Missing functionality] `stripWidth = chartW ?? cardW` fallback**
- **Found during:** Task 1 (svelte-check)
- **Issue:** `computeChartWidth` returns `number | undefined` (undefined when bars fit without horizontal scroll). Plan's `chartW / chartData.length` math would crash on undefined; passing `width={chartW}` would set the strip's width to `undefined`.
- **Fix:** Added `const stripWidth = $derived(chartW ?? cardW)` and used it for both eventBuckets pixel math and the `<EventBadgeStrip width={...}>` prop.
- **Files modified:** All 5 wired cards.
- **Commits:** `45a1141`, `28006d9`, `cce88d5`

**4. [Rule 1 — Type tightening] `EventBadgeStrip` events prop loosened to `readonly ForecastEvent[]`**
- **Found during:** Task 1 (the strip's prop was `ForecastEvent[]` mutable; `overlay.events` getter is `readonly ForecastEvent[]` per Plan 16.3-06)
- **Issue:** Type mismatch on assignment.
- **Fix:** Widened the strip's events prop to `readonly ForecastEvent[]`. Component reads only — no mutation, so the readonly contract is honest.
- **Files modified:** `EventBadgeStrip.svelte`
- **Commit:** `45a1141`

**5. [Rule 2] `CalendarItems*` events fetch uses card grain instead of hardcoded `day`**
- **Found during:** Task 2 (the plan suggested `granularity=day` hardcoded)
- **Issue:** When the user switches to week or month grain, hardcoded day-grain events would not match the card's bucket boundaries.
- **Fix:** Pass `granularity=${getFilters().grain}` so server returns events bucketed at the card's actual grain.
- **Files modified:** `CalendarItemsCard.svelte`, `CalendarItemRevenueCard.svelte`
- **Commit:** `28006d9`
