## Auto-flip directive

> **Auto-flip on right-edge overflow is handled by the parent wrapper's built-in `contained="window"` prop (CalendarRevenueCard precedent).**

The popup component itself does NOT manage edge detection. The parent LayerChart `Tooltip.Root` (or equivalent positioning wrapper) MUST be invoked with `contained="window"` so that on right-edge / top-edge overflow, the wrapper flips the popup to the opposite side of the anchor automatically. This behavior is preserved verbatim from `ForecastHoverPopup.svelte`'s host pattern in `RevenueForecastCard.svelte`.

Plan 16.3-03's popup util MUST document this `contained="window"` requirement in its docstring so per-card consumers (`CalendarRevenueCard`, `CalendarCountsCard`, `CalendarItemsCard`, `CalendarItemRevenueCard`, `RepeaterCohortCountCard`) wire the prop correctly when mounting the new `EventBadgeStrip` popup.
