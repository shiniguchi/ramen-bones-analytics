## Pattern: replaced single-row consumer with multi-row Tooltip.List

Inherited the working CalendarRevenueCard pattern. Tooltip.Root snippet body uses `{#snippet children({ data })}` (C-01 contract). Body computes `bucketIso` from `data.target_date_d`, then walks `splitSeriesByModel.entries()` filtering by `visibleModels.has(name)` and finding the matching row by exact ISO date in past-or-future arrays. Renders `<Tooltip.Header>` + `<Tooltip.List>` with one `<Tooltip.Item>` per visible model that has data at this bucket.

## Highlight binding strategy chosen

**Per-model loop** (preferred path per Plan Task 2). One `<Highlight>` instance per visible model, each bound to its own `modelData` (concatenated past + future rows for that model_name). LayerChart Highlight's `points` prop accepts a `Partial<ComponentProps<Circle>>` — passing `points={{ fill: FORECAST_MODEL_COLORS[modelName] }}` colors each dot per model. The chart-level `<Highlight points lines />` was REMOVED so the bisect-x default no longer drifts dots off-line.

`color` is NOT a top-level prop on LayerChart Highlight. Initial implementation used `color={FORECAST_MODEL_COLORS[modelName]}` and svelte-check rejected it: `Object literal may only specify known properties, and '"color"' does not exist in type 'HighlightPropsWithoutHTML'`. Fixed to use the documented `points={{ fill: ... }}` Circle-config form.
