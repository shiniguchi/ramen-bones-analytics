## Per-Spline Highlight dot binding

Each per-model Highlight is bound to its own `modelData` (concatenated past+future arrays for that model_name) — `<Highlight data={modelData} ... points={{ fill: FORECAST_MODEL_COLORS[modelName] }} lines={false} />` inside the `{#each Array.from(splitSeriesByModel.entries()) ...}` block. Dots use the model's palette color via the `points` Circle config (LayerChart Highlight does not accept a top-level `color` prop — it expects a Circle config under `points`).

The chart-level `<Highlight points lines />` was removed from both files. Per-Spline binding ensures each model's highlight dot follows ITS spline path (D-05) rather than the chart's bisect-x default which drifted dots off-line for non-leading models.
