# 16.2-03 SUMMARY — Calendar* tooltip layout + Spline z-order verify

## Item 3 — Tooltip model row layout fix

**Pattern:** hand-rolled `<li class="flex items-center justify-between gap-3 py-0.5 text-xs">` row replaces the `<Tooltip.Item>` invocation for model rows. Cohort rows (1st/2nd/3rd/...Total) above the divider continue using `<Tooltip.Item>` unchanged.

The new markup:
- `flex items-center justify-between gap-3` — label-LEFT value-RIGHT, single row
- Inner span on label: `min-w-0` to allow shrink + `truncate` so overflowing labels ellipsis instead of wrapping
- Outer span on value: `flex-shrink-0 whitespace-nowrap tabular-nums` so the value never wraps and never gets pushed below the label
- Color dot: `<span class="inline-block h-2 w-2 flex-shrink-0 rounded-full" style:background-color={...}>` matching the model's palette color

This addresses the owner-reported regression where `<Tooltip.Item>`'s default layout broke when a model label (e.g., "Naive (DoW avg)") wrapped at narrow viewports — pushing the value to a new visual row and effectively flipping the apparent row order.
