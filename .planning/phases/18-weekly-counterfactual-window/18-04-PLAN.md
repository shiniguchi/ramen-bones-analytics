Keep the existing 7-key `uplift_hero_*` i18n matrix and the maturity-tier × CI-overlap selection logic, but switch the maturity-tier source per **Decision A (option a — locked here)**:

- The existing function maps `n_days < 14 → early`, `< 28 → midweeks`, else `mature`. With per-week reads, `n_days = 7` always for fully-completed weeks, so every weekly hero would resolve to `early` → `uplift_hero_too_early`. Wrong.

const maturityTier = $derived.by<MaturityTier>(() => {
  if (!headline?.campaign?.start_date) return 'early';
  const startMs = parseISO(headline.campaign.start_date).getTime();
  const todayMs = Date.now();
  const weeksSinceLaunch = Math.floor((todayMs - startMs) / (7 * 24 * 60 * 60 * 1000));
  if (weeksSinceLaunch < 2) return 'early';
  if (weeksSinceLaunch < 4) return 'midweeks';
  return 'mature';
});
