<interfaces>
From Plan 03 the loader now returns `data.filters: FiltersState`, `data.distinctSalesTypes: string[]`, `data.distinctPaymentMethods: string[]`, and `data.window: RangeWindow` in addition to the existing `range` / `grain`.

`FiltersState` shape:
```ts
{
  range: 'today' | '7d' | '30d' | '90d' | 'all' | 'custom';
  grain: 'day' | 'week' | 'month';
  sales_type: string[] | undefined;   // ['INHOUSE'] | ['INHOUSE','TAKEAWAY'] | undefined
  payment_method: string[] | undefined;
  from: string | undefined;   // 'YYYY-MM-DD'
  to: string | undefined;
}
```

URL convention (D-16 keep defaults; D-12 omit 'All'):
- preset range: `?range=7d&grain=week`
- custom range: `?range=custom&from=2026-04-01&to=2026-04-15&grain=week`
- multi-select: `?sales_type=INHOUSE,TAKEAWAY&payment_method=visa,mastercard`
- 'All' sentinel: omit the param entirely

SvelteKit `goto()` from `$app/navigation`; current page URL from `$app/state` page.url per 04-CONTEXT.
