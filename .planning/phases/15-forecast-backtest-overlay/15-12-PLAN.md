# Phase 15-12: `CalendarRevenueCard` Overlay — Forecast Lines + CI Bands on Bars

> REQUIRED SUB-SKILL: superpowers:subagent-driven-development.

**Goal:** Extend `CalendarRevenueCard.svelte` to render per-model forecast LINES (Spline) + low-opacity CI BANDS (Area) on top of the existing visit-seq stacked bars. Add inline `ForecastLegend` chip row. X-axis extends to `last_actual + 365d` with horizontal scroll. Default visible models: `sarimax` + `naive_dow`.

**Architecture:** Self-fetch via `clientFetch('/api/forecast?kpi=revenue_eur&granularity=...')` keyed off the global `getFilters().grain`. Render inside the existing `<Chart>` block: bars first (existing), then `<Area>` per visible model for CI band, then `<Spline>` per visible model for line. Switch from band scale to **time scale** for the X-axis to support the extended forward range; bars retain `bandwidth` calculated from `xScale(addDays(d, 1)) - xScale(d)`.

**Tech Stack:** Svelte 5 runes, LayerChart 2.x (`Chart`/`Bars`/`Area`/`Spline`), date-fns.

### Task 1: Extend `CalendarRevenueCard.svelte`

- [ ] **Step 1: Read current `src/lib/components/CalendarRevenueCard.svelte`** (post-squash) to understand the existing visit-seq stacked-bar shape.

- [ ] **Step 2: Add forecast state + fetch**:

```svelte
<script lang="ts">
  // ... existing imports ...
  import { clientFetch } from '$lib/clientFetch';
  import ForecastLegend from './ForecastLegend.svelte';
  import { FORECAST_MODEL_COLORS } from '$lib/chartPalettes';
  import { Area, Spline } from 'layerchart';
  import { addDays, parseISO } from 'date-fns';

  // ... existing visit-seq state ...

  // Forecast overlay state
  type ForecastRow = { target_date: string; model_name: string; yhat_mean: number; yhat_lower: number; yhat_upper: number };
  type ForecastPayload = { rows: ForecastRow[]; actuals: { date: string; value: number }[]; events: unknown[]; last_run: string | null };

  let forecastData = $state<ForecastPayload | null>(null);
  let visibleModels = $state(new Set<string>(['sarimax', 'naive_dow']));

  // Re-fetch on grain change. Plain JS guard prevents reactive loop.
  let lastFetchedGrain: string | null = null;
  $effect(() => {
    const g = getFilters().grain;
    if (g === lastFetchedGrain) return;
    lastFetchedGrain = g;
    void clientFetch<ForecastPayload>(`/api/forecast?kpi=revenue_eur&granularity=${g}`)
      .then(f => { forecastData = f; })
      .catch(e => console.error('[CalendarRevenueCard /api/forecast]', e));
  });

  function toggleModel(m: string) {
    const next = new Set(visibleModels);
    next.has(m) ? next.delete(m) : next.add(m);
    visibleModels = next;
  }

  // Per-model row arrays for visible models only.
  const seriesByModel = $derived.by(() => {
    if (!forecastData) return new Map<string, ForecastRow[]>();
    const map = new Map<string, ForecastRow[]>();
    for (const r of forecastData.rows) {
      if (!visibleModels.has(r.model_name)) continue;
      if (!map.has(r.model_name)) map.set(r.model_name, []);
      map.get(r.model_name)!.push(r);
    }
    for (const arr of map.values()) {
      arr.sort((a, b) => a.target_date.localeCompare(b.target_date));
    }
    return map;
  });

  // Available models from current payload (drives ForecastLegend disabled-state).
  const availableModels = $derived(Array.from(new Set(forecastData?.rows.map(r => r.model_name) ?? [])));
</script>
```

- [ ] **Step 3: Inside the existing `<Chart>` block, ADD overlay markup BELOW the existing `<Bars>` block**:

```svelte
<Svg>
  <!-- existing axes + bars + trendline + value labels... -->

  <!-- Forecast CI bands (back of overlay layer, low opacity) -->
  {#each Array.from(seriesByModel.entries()) as [modelName, modelRows] (modelName + '-band')}
    <Area
      data={modelRows.map(r => ({ ...r, d: parseISO(r.target_date) }))}
      x={(r: { d: Date }) => r.d}
      y0={(r: { yhat_lower: number }) => r.yhat_lower / 100}
      y1={(r: { yhat_upper: number }) => r.yhat_upper / 100}
      fill={FORECAST_MODEL_COLORS[modelName]}
      fillOpacity={0.06}
    />
  {/each}

  <!-- Forecast lines (front of overlay layer) -->
  {#each Array.from(seriesByModel.entries()) as [modelName, modelRows] (modelName + '-line')}
    {@const isNaive = modelName === 'naive_dow'}
    <Spline
      data={modelRows.map(r => ({ ...r, d: parseISO(r.target_date) }))}
      x={(r: { d: Date }) => r.d}
      y={(r: { yhat_mean: number }) => r.yhat_mean / 100}
      stroke={FORECAST_MODEL_COLORS[modelName]}
      stroke-width={isNaive ? 1 : 2}
      stroke-dasharray={isNaive ? '4 4' : undefined}
    />
  {/each}
</Svg>
```

> **Note on the `/100` divisions** above: bars are in cents (`Math.round(v / 100)` is done in chartData mapping); forecast yhat values from `/api/forecast?kpi=revenue_eur` come in EUROS (per Phase 14 `forecast_daily.yhat`). To overlay correctly we render forecasts in EUR matching the bars' EUR display. **Key constraint**: ensure y-axis domain accommodates BOTH bar tops AND forecast yhat_upper extents. Compute `yDomain` to include `max(bar_total, forecast yhat_upper)`.

- [ ] **Step 5: Update `xDomain` and `chartData` to extend to `last_actual + 365d`**.

The existing card uses `bucketRange(w.from, w.to, grain)` to enumerate periods. Extend the upper bound to include forecast horizon. Use a separate `chartXDomain = $derived([from, addDays(today, 365)])` and pass to `<Chart xDomain={chartXDomain}>`.

- [ ] **Step 6: Switch X-axis from band scale to time scale**.

Replace `xScale={scaleBand()}` (current implicit) with `xScale={scaleTime()}`. Bars need explicit width: compute `bandwidth = (xScale(addDays(d, 1)) - xScale(d)) * 0.8` per bar. LayerChart's `<Bars>` accepts a `bandwidth` prop on time-scale X.
