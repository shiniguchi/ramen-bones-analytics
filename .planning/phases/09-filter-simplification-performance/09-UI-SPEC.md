## Layout Contract

### Filter Bar (2-row, sticky)

```
Row 1: [DatePickerPopover]
Row 2: [GrainToggle] | [SalesType: All/Inhouse/Takeaway] | [CashCard: All/Cash/Card]
        ^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^
        horizontally scrollable on narrow screens, no wrapping
```

**Row 1 spec:**
- Full width, contains DatePickerPopover trigger button only
- Same pattern as current FilterBar row 1

**Row 2 spec:**
- `overflow-x: auto` with `-webkit-overflow-scrolling: touch`
- `flex-nowrap` -- toggles never wrap to a third row
- Vertical 1px separators (`border-l border-zinc-200 h-6 self-center`) between each toggle group
- Gap between separator and adjacent toggles: 8px (`gap-2`)
- No horizontal scrollbar visible (use `scrollbar-width: none` / `::-webkit-scrollbar { display: none }`)

**Sticky behavior:**
- `sticky top-0 z-30` (same as current)
- `border-b bg-background/95 backdrop-blur` (same as current)
- Max height budget: 80px (current is 72px; row 2 toggles add ~8px)

### KPI Tiles (2 tiles)

```
[Revenue . {range}]  [Transactions . {range}]
```

- 2 tiles in a `grid grid-cols-2 gap-4` layout (same pattern as existing, just fewer tiles)
- Each tile: `rounded-xl border border-zinc-200 bg-white p-4`
- Title: dynamic, includes range label (e.g., "Revenue . 7d", "Transactions . 30d")
- Delta line: shows `vs prior {range}` or "-- no prior data" for range=all

## Interaction Contract

### Segmented Toggle Interaction

| Action | Result | Latency Target |
|--------|--------|----------------|
| Tap segment | Active segment highlights (blue-50 bg, blue-600 text). URL param updates via `replaceState()`. Client-side data re-filters/re-buckets. | <50ms visual, <200ms data |
| Tap already-active segment | No-op | 0ms |

ARIA: `role="group"` on container, `role="radio"` + `aria-checked` on each segment.

### Date Range Change

| Action | Result | Latency Target |
|--------|--------|----------------|
| Tap preset chip (7d/30d/etc) | URL updates via `replaceState()`. If cache covers range, instant slice. If not, client-side fetch then slice. Popover closes. | <50ms if cached, <500ms if fetch needed |
| Apply custom range | Same as preset but with from/to params | <500ms |

### Grain Change

| Action | Result | Latency Target |
|--------|--------|----------------|
| Tap Day/Week/Month | Client-side re-bucketing of already-loaded daily rows. URL updates via `replaceState()`. | <200ms (VA-12 requirement) |

### Filter Combination

All three toggles (grain, sales_type, is_cash) compose additively. Changing any one re-derives the visible dataset from raw daily rows. No server round-trip for any combination change.

## State Management Contract

| State | Location | Sync Mechanism |
|-------|----------|----------------|
| `range` | URL param `?range=` | `replaceState()` |
| `grain` | URL param `?grain=` | `replaceState()` |
| `sales_type` | URL param `?sales_type=` | `replaceState()` |
| `is_cash` | URL param `?is_cash=` | `replaceState()` |
| `from` / `to` | URL params (custom range only) | `replaceState()` |
| Raw daily rows | `dashboardStore.svelte.ts` `$state` | SSR initial load, client-side fetch on range widen |
| Filtered+bucketed data | `dashboardStore.svelte.ts` `$derived` | Reactive from raw rows + filter params |
| Widest cached window | `dashboardStore.svelte.ts` `$state` | Updated on each fetch |
