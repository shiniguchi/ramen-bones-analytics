## Overview

Single-feature milestone replacing the `CampaignUpliftCard`'s cumulative-since-launch headline with a per-ISO-week (Mon–Sun) counterfactual answer plus a tap-scrubbable bar-chart history of all completed weeks since campaign launch. The friend-owner now gets a fresh weekly read on whether the campaign is working — "Week of Apr 27 – May 3: −€149" — instead of a single decaying cumulative number that drifts toward "no detectable lift" the longer the campaign runs.

The delivery touches every layer of the stack: DB schema (new `window_kind = 'iso_week'` + wrapper view), forecast pipeline (independent bootstrap CI re-fit per 7-day slice), API (new `weekly_history[]` field), Svelte component (hero rewrite + LayerChart bar chart), and i18n (3 new keys × 5 locales).

## Milestone Summary

**Key Decisions:**

- **Decision A (LOCKED):** Maturity tier from `Math.floor((today − campaign.start_date) / 7 weeks)`, not `n_days`. Per-week rows always have `n_days=7` → would always resolve to "early" under the old n_days-based derivation. Fix anchors tier to elapsed campaign duration.
- **Decision B PRIMARY → Option C fallback:** Three filtered `<Bars>` blocks (Decision B) computed independent band-scale domains → NaN x/width on all bars. Fell through to Option C: manual `<rect>` loop via `chartCtx.xScale` from full `weeklyHistory` domain. Context7 query confirmed no per-bar render snippet API in LayerChart 2.x stable.
- **Overflow fix:** `style:overflow="hidden"` inline wins over LayerChart's component-scoped `overflow: visible` on `lc-layout-svg`. Tailwind `overflow-hidden` was silently losing. CI whiskers for a ±€2,000 range on a 100px chart bled ~1,600px without the fix.
- **Bootstrap CI independence:** Each weekly row bootstrapped from its own 7-day residuals — NOT derived by subtracting daily CI bounds (correlated samples, anti-pattern documented in CONTEXT.md).
- **Backwards-compat:** `cumulative_since_launch` rows continue to be written by pipeline; API top-level cumulative fields preserved. Cleanup is a future phase.
- **Seed namespace:** Per-week 100_000+k, per-day 42+i — disjoint, documented as forward-compat invariant.

**Issues Resolved:**

- Overflow bleed: CI whiskers spanning ±€2,000 on a 100px chart bled ~1,600px above the card (commit fb97843)
- Decision B NaN bars: independent domain per `<Bars>` subset → Option C manual `<rect>` (commit 90fba8e)
- `yDomain` visibility: bars invisible when only point_eur used in domain calculation (commit 8b31212)

**Issues Deferred:**

- Cumulative-since-launch row removal from DB + API: future cleanup phase
- Per-day CI bands on historic sparkline: out of scope per CONTEXT.md
- Multi-campaign weekly chart: single-campaign assumption preserved
- Backfill of weekly history into prior `as_of_date` snapshots: audit trail begins from first run forward
- `de`/`es`/`fr` real translations for new i18n keys: v1.4 polish backlog

**Technical Debt Incurred:**

- `onBarClick` camelCase vs `onbarclick` — verified in LayerChart source, but not in docs; could silently break on LayerChart upgrade
- `chartCtx.xScale` access pattern (Option C) is an internal LayerChart API; watch for breakage on upgrades
