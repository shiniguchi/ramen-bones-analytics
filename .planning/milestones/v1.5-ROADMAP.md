## Overview

Single-phase performance milestone that eliminates the three primary cold-start bundle blockers so the dashboard paints above-fold KPI tiles without downloading LayerChart (4.9 MB), the full i18n monolith (76 KB), or firing benchmark/item-count queries at SSR time.

All three axes were addressed atomically in one feature branch (`feature/phase-19-cold-start-trim`), shipped in ~1.5 hours. 30 async chunks visible in the CF Pages build output.

## Phases

### Phase 19: Cold-Start Trim

**Goal:** Eliminate the three primary cold-start bundle blockers so the dashboard paints above-fold KPI tiles without downloading LayerChart (4.9 MB), the full i18n monolith (76 KB), or firing benchmark/item-count queries at SSR.

**Depends on:** Phase 18 (CampaignUpliftCard already lazy-wrapped; deferred /api/* pattern established in Phase 11)

**Requirements:** PERF-01, PERF-02, PERF-03

**Success Criteria** (verified ✅):
1. Cold load shows skeleton placeholders for 5 deferred chart cards; KPI tiles paint without LayerChart in the critical path ✅
2. Scrolling to each deferred card triggers its async chunk download (visible in Network panel filtered to `chunk-`) ✅
3. SSR `Promise.all` reduced from 6 to 3 (exceeded SC target of 4); `/api/item-counts` and `/api/benchmark` return correct data ✅
4. `messages.ts` on disk ≤3 KB; 4 async locale chunks visible in build output; locale switch renders correct strings ✅ (3.6 KB achieved)
5. `npm run check` + `npm run test:unit` exit 0 ✅

**Plans:**

- [x] 19-01-PLAN.md — LazyMount loader prop + 9 eager chart cards deferred (CampaignUpliftCard, DailyHeatmapCard, CalendarCountsCard, CalendarRevenueCard, CalendarItemsCard, CalendarItemRevenueCard, CohortRetentionCard, RepeaterCohortCountCard, MdeCurveCard)
- [x] 19-02-PLAN.md — `/api/item-counts` + `/api/benchmark` deferred endpoints; SSR Promise.all 6→3
- [x] 19-03-PLAN.md — i18n per-locale dynamic imports (5 dict files + `loadDict()` lazy cache; messages.ts 76 KB → 3.6 KB)
- [x] 19-04-PLAN.md — phase-final QA + NaN tooltip band rect fix + planning-docs drift gate pass

**Key implementation notes:**
- `loader` prop on `LazyMount.svelte` accepts `() => Promise<{ default: Component }>` — Intersection Observer fires the import only when the card scrolls into view
- `{@const DynComp = Loaded}` pattern used (not `<svelte:component this={...}>` — deprecated in Svelte 5 runes mode)
- `loadDict()` uses switch-case (not template literal) for Vite static analysis to emit per-locale chunks
- `messages.en` compatibility shim retained (deprecated) — 3 test files import it; cleanup deferred
